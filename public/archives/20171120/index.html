横向き印刷対応（landscape print） in 2017

2017/11/20

  • print

印刷対応の話って、正直あんまり書きたくないんですよ。なんでですかね？

でもメモっておかないとせっかく調べたことが報われないというか、 もし近い将来誰かが同じようなことを調べようとしたりしたら、 それこそ時間の無駄なので、重い腰を上げてメモしておこうと思います。

印刷の向きについて

まず基本的な事柄から。（ただしあまり昔話はしない方向で。）

印刷の向きについては、もともとは縦が基本でした。 PC で見たときの幅に収める作りから、可変幅の作りに変わりつつある中で、 印刷時における向きの指定を CSS で出来るようになってきて、 ブラウザによっては CSS の指定だけで横向き印刷ができるようになってきています。

また、これもブラウザによってはなのですが、 CSS の指定は有効になっていつつも、最終的な印刷の向きは印刷時の設定画面にて行うブラウザもあります。

ややこしいので現状をまとめてみます。

  • Chrome: CSS で横向き（landscape）指定してあると、そのまま横向き印刷できる
  • レイアウト的に横向き印刷対応してあるが、印刷時に縦横を選択する - Firefox
    • 2回目以降の印刷は、前回の印刷向きを踏襲する
  • 印刷時は縦がデフォルトだが、印刷プレビュー時は縦横選択できる - IE11
    • JavaScript から印刷プレビュー画面を直接開くことはほぼ不可能、ActiveX でのコントロールが必要（≒セキュリティレベルを下げるなどの対応が必要）

  • 印刷領域に height: 100vh とかつけてると、IE11 のみ最後に白紙のページが残る、ぎりぎりはみ出るらしい
  • そういえば page-break-after が効かなかった、なぜ・・・

以下、70.55% でピタッと収まる数字的な根拠が欲しい。

.container-parent {
  width: 100vw;
  height: 99.9vh; // unnecessary last page in ie '100vh'
  box-sizing: border-box;
  overflow: hidden;
  page-break-after: always;
}

.container-child {
  width: 100vh;
  height: 100vw;
  position: relative;
  transform-origin: 50% 70.55%;
  transform: rotate(-90deg);
}
